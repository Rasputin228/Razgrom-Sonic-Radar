# Razgrom Sonic Radar V5.1

Аудио-радар для визуализации звука в играх. Проект ориентирован на глухих и слабослышащих пользователей: он показывает направление и примерную силу звука на прозрачном оверлее поверх выбранного окна.

![Status](https://img.shields.io/badge/Status-Active-green) ![License](https://img.shields.io/badge/License-MIT-blue)

## ⚠️ Отказ от ответственности и Риски

**Используйте на свой страх и риск.**

1.  **Политика игр:** В большинстве соревновательных игр (CS2, Arena Breakout, Tarkov) использование сторонних оверлеев, дающих преимущество, может расцениваться как нарушение правил.
2.  **Античит:** Программа не внедряется в память игры (No Injection), не читает память процесса игры и работает только с аудио-потоком Windows. Это инструмент доступности (Accessibility Tool), но античиты могут реагировать на любые оверлеи. Проект не содержит функций обхода античита.
3.  **Автор не несет ответственности** за возможные блокировки аккаунтов.

## 📦 Установка и запуск

1.  Скачайте файл `RazgromOverlay.exe` из раздела **Releases**.
2.  Запустите файл от имени **Администратора**.
3.  В открывшемся окне конфигуратора:
    *   Выберите окно игры из списка.
    *   Выберите источник звука.
    *   Настройте чувствительность, размер, прозрачность и цветовой режим.
4.  Нажмите **ЗАПУСТИТЬ**.

### Быстрый запуск из исходников

В папке проекта есть `start_radar.bat`. Он показывает консольный баннер из `assets/banner.txt`, проверяет Python и запускает основной оверлей:

```powershell
start_radar.bat
```

Если проект лежит внутри общей папки `RazgromRadar`, можно запускать батник из верхней папки: `Start_Razgrom_Radar.bat`.

### Управление
*   **Перемещение:** Зажмите левую кнопку мыши на радаре.
*   **Размер:** Потяните за треугольник в правом нижнем углу.
*   **Меню:** Нажмите правой кнопкой мыши по радару.
*   **Закрытие:** Правый клик по радару -> `Закрыть радар`.
*   **Тест направлений:** Правый клик по радару -> `Тест направлений`, затем выберите сторону.

### Настройки доступности
*   **Чувствительность:** Увеличьте, если тихие шаги или действия не видны. Уменьшите, если радар постоянно светится от фонового шума.
*   **Размер радара:** Подберите размер под монитор и игру. Его также можно менять за угол в самом оверлее.
*   **Непрозрачность:** Делает радар менее или более заметным поверх игры.
*   **Цвета:** Доступны оранжевый, сине-белый и высококонтрастный режим.
*   **Подписи направлений:** `F`, `R`, `B`, `L` помогают быстрее понять сторону источника звука.
*   **Статус:** `IDLE` - тишина, `SOUND` - обнаружен звук, `MOVE` - пользователь двигается и шум подавляется, `LOUD` - сильный звук.
*   **Ширина сектора:** Управляет тем, насколько широко загорается направление вокруг основного сектора.
*   **Сглаживание:** Управляет стабильностью направления. Меньше значение - плавнее, больше - быстрее реакция.
*   **Поменять левый/правый канал:** Используйте, если тестовый звук слева отображается справа или наоборот.
*   **Порог шума:** Минимальный уровень сигнала, ниже которого радар считает звук фоном.
*   **Калибровать тишину:** Измеряет фон выбранного аудиоисточника за 3 секунды и автоматически выставляет порог шума.
*   **Профиль:** Быстрый пресет под игру или режим проверки. Можно выбрать `Custom`, `CS2 / footsteps`, `Tarkov / tactical`, `Arena Breakout`, `Desktop test`.
*   **Диагностика источника:** За 1 секунду показывает средний уровень, пик, баланс лево/право и частотный центр выбранного аудиоисточника.
*   **Preflight перед запуском:** При нажатии `ЗАПУСТИТЬ РАДАР` выбранный источник проверяется до открытия оверлея. Если устройство не открывается, запуск не продолжается.
*   **Диагностический отчет:** Сохраняет `diagnostic_report.json` с выбранным устройством, настройками, последним измерением и списком доступных источников.
*   **Режим HUD:** `radar` показывает полный круговой радар, `minimal` оставляет компактные секторные вспышки без лишней сетки.
*   **Индикаторы по краям:** Дополнительные полосы сверху, справа, снизу и слева помогают замечать направление периферийным зрением.
*   **CONF:** Процент уверенности в текущем направлении. Низкое значение означает, что сигнал слабый или почти одинаковый в двух каналах.
*   **EVENT:** Простая классификация текущего звука: `STEP`, `IMPACT`, `SHARP`, `LOW`, `SOUND`, `IDLE`.
*   **AUDIO ERROR:** Если выбранное аудиоустройство отвалилось или не открылось, HUD покажет ошибку, а подробности попадут в `overlay_debug.txt`.

Позиция и размер оверлея сохраняются автоматически после перемещения или изменения размера.

### Геометрия радара

Оверлей использует 16 секторов по 22.5 градуса:

*   `0` - вперед.
*   `4` - вправо.
*   `8` - назад.
*   `12` - влево.

Передние и задние направления рассчитываются отдельно. Для задней полусферы левый и правый канал не зеркалятся: задний правый звук уходит в правую половину радара, задний левый - в левую.

## ⚙️ Настройка звука (Изоляция)

Для корректной работы радара рекомендуется использовать виртуальный аудио-кабель, чтобы исключить посторонние звуки (Discord, музыка).

1.  Установите драйвер **VB-CABLE**.
2.  В настройках Windows перенаправьте звук игры на `CABLE Input`.
3.  В настройках записи звука (`mmsys.cpl`) для `CABLE Output` включите "Прослушивать с данного устройства" на ваши наушники.
4.  В программе выберите `CABLE Output`.

### Выбор источника

В конфигураторе доступны два типа источников:

*   `OUTPUT LOOPBACK` - звук, который Windows отправляет на выбранные динамики, наушники или виртуальный кабель. Это основной режим для игр.
*   `MIC INPUT` - обычный вход микрофона. Используйте только если нужно визуализировать внешний звук с микрофона.

Чтобы Discord, браузер, музыка или микрофон не перекрывали игровой звук, выводите игру на отдельное устройство или `VB-CABLE`, а остальные приложения оставляйте на другом устройстве. Затем выберите нужный `OUTPUT LOOPBACK` в программе.

## 🛠 Запуск из исходников

```powershell
pip install -r requirements.txt
python Overlay\main_overlay.py
```

Проверка математики направлений:

```powershell
python -m unittest discover -s tests
```

Для сборки EXE можно использовать существующий `RazgromOverlay.spec` из корня проекта:

```powershell
pyinstaller ..\RazgromOverlay.spec
```

### Структура overlay-кода

*   `main_overlay.py` - лаунчер, оверлей и связка аудиодвижка с UI.
*   `app_config.py` - настройки, профили, сохранение `config.json`.
*   `audio_io.py` - выбор источников, диагностика, preflight и диагностический отчет.
*   `audio_direction.py` - математика направлений и сглаживание углов.
*   `audio_events.py` - классификация событий `STEP`, `IMPACT`, `SHARP`, `LOW`.
